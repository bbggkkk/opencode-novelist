# Production Readiness Audit

This audit maps the core novelist-agent requirements to concrete repository evidence and validation gates. It is intended as the release checklist for changes that affect writing, revision, verification, publishing, installation, or templates.

## Release Gate

Run:

```bash
make validate
```

Expected checks:

- `agent frontmatter ok`
- `continuity scenario ok`
- `style and character drift scenario ok`
- `verification manifest ok`
- `sample work fixture ok`
- `production invariants ok`
- `verification manifest negative tests ok`
- `install smoke ok`
- `all validations ok`

## Known Boundary

The validation suite proves repository-level invariants, sample fixtures, installer packaging, and failure detection for manifest publication gates. It cannot prove that every future model response will be perfect in arbitrary user projects, so the agent design enforces strict loops, durable artifacts, Otaku verification, and publication gates to reduce unsafe delivery paths rather than relying on a single generation.
