# 모델 추천 가이드

이 문서는 Novelist 에이전트별 추천 모델 역량을 정리합니다. 아래에 적힌 모델명은 필수 조건이 아니라 예시입니다. 모델명, 제공사 라인업, 실제 서빙 품질은 계속 바뀌므로 오래 유지되어야 하는 기준은 각 역할에 필요한 역량 프로필입니다.

영문 문서: [model-recommendations.md](model-recommendations.md)

## 일반 선택 원칙

아키텍처 이름보다 역할 적합성을 우선합니다. "Dense"와 "MoE"는 유용한 약어일 수 있지만, 제공사가 세부 구조를 항상 공개하지는 않으며 실제 런타임 품질이 더 중요합니다.

- Writer와 Editor는 고품질 dense 또는 dense-like 창작 모델을 우선합니다. 최대 도구 사용 능력보다 안정적인 문장 리듬, 캐릭터 어투 유지, 문체 제어가 중요합니다.
- Router, Otaku, Loremaster, Researcher, Publisher는 긴 문맥 안정성이 좋은 대형 추론 모델을 우선합니다. exhaustive verification, 검색/취합, 구조화 추론, 도구 사용이 강하다면 MoE 모델도 적합합니다.
- 판단이 애매하면 가장 강한 추론 모델을 `novelist-otaku`와 `novelist`에 먼저 배정하고, 그다음 가장 좋은 산문 모델을 `novelist-writer`에 배정합니다.

## 예시 모델 조합

다음 조합은 현재 테스트 선호를 반영한 예시입니다.

| 역할 그룹 | 예시 모델 |
|-----------|-----------|
| 창작 산문 및 미시 편집 | `gemma 4 31b` |
| 라우팅, 검증, 검색/취합, 조사, 출판 | `deepseek v4 flash` |

이 조합은 시작점입니다. 로컬 벤치마크에서 한국어 산문, 긴 문맥 검증, XML/파일 생성에 더 좋은 모델이 확인되면 벤치마크 결과를 우선합니다.

## `novelist` 라우터

라우터는 사용자 요청을 분류하고, 요청 범위를 고정하며, Seed-to-Fruit 파이프라인을 관리하고, 하위 에이전트를 호출하며, 완료 게이트를 강제합니다.

세부 요구사항:
- 지시 준수와 라우팅 판단이 강해야 합니다.
- `writing-session.md`, `verification-manifest.md`, `narrative-state.md`, 사용자 요청을 오가는 긴 문맥 추론이 안정적이어야 합니다.
- 중단/계속 판단, 재개 검증, 레트콘 분류, 파이프라인 완료 감사에 대한 판단력이 좋아야 합니다.
- 필수 단계를 건너뛰려는 단축 요청을 엄격히 거부해야 합니다.

추천 모델 프로필:
- 대형 추론 모델.
- 도구 사용과 구조화 출력이 강한 모델.
- 긴 문맥 안정성이 좋다면 MoE도 허용됩니다.

예시 모델:
- `deepseek v4 flash`

## `novelist-writer`

Writer는 실제 소설 문장을 작성합니다. 장면, 서술, 대사, 감정 이동, Macro Skeleton에 붙는 비트 단위 산문을 만듭니다.

세부 요구사항:
- 한국어 문장 리듬과 대사가 좋아야 합니다.
- 긴 세션에서도 캐릭터 어투와 톤이 안정적으로 유지되어야 합니다.
- 누적된 검증 완료 prefix에서 자연스럽게 이어 쓰는 능력이 강해야 합니다.
- 캐논이나 가지의 종점을 몰래 바꾸지 않는 창작 판단력이 필요합니다.
- 외부 도구 사용 능력보다 산문 품질이 더 중요합니다.

추천 모델 프로필:
- 고품질 dense 또는 dense-like 창작 모델.
- 중간 이상에서 큰 문맥 창.
- 검증 에이전트보다 높은 창의성 허용치.

예시 모델:
- `gemma 4 31b`

## `novelist-editor`

Editor는 미시적 문장 및 장면 편집자입니다. 문체, 어투, 서식, 로컬 인과, 페이스, 즉각적인 가독성을 다듬습니다.

세부 요구사항:
- 산문 편집과 문체 제어가 강해야 합니다.
- 한국어 문장 리듬, 대사 다듬기, 서식 규율이 좋아야 합니다.
- 잠긴 prefix와 수정 가능 구간 경계를 보존해야 합니다.
- Macro Skeleton은 로컬 가드레일로만 사용하며 전체 흐름 판정은 맡지 않습니다.
- 문체, 어투, macro guardrail 메모를 명확히 남겨야 합니다.

추천 모델 프로필:
- 고품질 dense 또는 dense-like 창작/편집 모델.
- Writer와 비슷한 급이 좋으며, 문체와 어투가 뛰어나다면 약간 작은 모델도 가능합니다.
- 가장 강한 전역 추론 모델일 필요는 없습니다.

예시 모델:
- `gemma 4 31b`

## `novelist-otaku`

Otaku는 설정, 연속성, 거시 흐름을 검증합니다. 모든 세부사항을 캐논과 대조하고, 병합 전 Branch Traversal Audit을 수행합니다.

세부 요구사항:
- 긴 문맥 추론과 집요한 세부 검토 능력이 강해야 합니다.
- 캐릭터 시트, 세계관 규칙, 시리즈 바이블, narrative state, 검증된 prefix, Macro Skeleton 사이의 모순을 안정적으로 찾아야 합니다.
- 증거 기반 검증 보고서를 작성해야 합니다.
- 단축 승인 없이 엄격한 PASS/FAIL을 유지해야 합니다.
- 가지 이탈, 셋업/페이오프 누락, 잘못된 캐릭터 지식, 소지품 드리프트, 타임라인 오류, 안전하지 않은 캐논 확장을 잡아낼 수 있어야 합니다.

추천 모델 프로필:
- 사용 가능한 가장 강한 추론 모델.
- 매우 좋은 긴 문맥 안정성.
- 검증 일관성이 높다면 MoE도 허용됩니다.

예시 모델:
- `deepseek v4 flash`

## `novelist-loremaster`

Loremaster는 캐논을 검색하고 정리합니다. 설정 파일, 시리즈 바이블, narrative state, 관련 산출물을 뒤지되 없는 사실을 invent하지 않습니다.

세부 요구사항:
- 검색 규율과 문맥 취합 능력이 강해야 합니다.
- 파일 경로와 줄 단위 인용이 좋아야 합니다.
- 누락되거나 서로 모순되는 원천 자료를 감지해야 합니다.
- 창의성은 낮고 원천 파일 충실도는 높아야 합니다.
- 큰 프로젝트 디렉터리에서도 긴 문맥 처리가 안정적이어야 합니다.

추천 모델 프로필:
- 대형 추론 또는 검색/취합 지향 모델.
- 구조화 요약과 인용 동작이 강한 모델.
- 원천 충실도가 높다면 MoE도 허용됩니다.

예시 모델:
- `deepseek v4 flash`

## `novelist-researcher`

Researcher는 현재 이야기, 시점, 문체, 캐논 제약을 기준으로 현실 세계 정보를 조사합니다.

세부 요구사항:
- 사실 추론과 출처 처리가 강해야 합니다.
- 원자료를 장면에 필요한 제약으로 걸러낼 수 있어야 합니다.
- "확인된 사실", "장면에 쓸 수 있는 정보", "본문 밖에 둬야 할 배경지식"을 잘 구분해야 합니다.
- 외부 잡지식으로 캐논을 덮어쓰려는 경향이 낮아야 합니다.

추천 모델 프로필:
- 도구가 있을 때 브라우징/출처 종합이 강한 추론 모델.
- 조사 결과를 캐논과 맞추기 위한 긴 문맥 안정성.

예시 모델:
- `deepseek v4 flash`

## `novelist-publisher`

Publisher는 수정 가능한 EPUB 원본을 만들고 검증된 초고를 패키징합니다. 이야기 문장을 쓰거나 수정하지 않아야 합니다.

세부 요구사항:
- 구조화된 파일 생성이 강해야 합니다.
- XML/XHTML/CSS/OPF/NCX 규율이 좋아야 합니다.
- zip 순서와 출판 게이트 지시를 안정적으로 따라야 합니다.
- manifest, 해시, 증거, 원본 검증이 실패하면 멈출 수 있어야 합니다.

추천 모델 프로필:
- 코드와 구조화 출력이 좋은 추론 모델.
- 최고의 창작 산문 모델일 필요는 없습니다.

예시 모델:
- `deepseek v4 flash`

## 실전 배정표

| 에이전트 | 핵심 필요 역량 | 추천 역량 티어 | 예시 |
|----------|----------------|----------------|------|
| `novelist` | 라우팅, 범위 고정, 파이프라인 강제 | 대형 추론 / 긴 문맥 | `deepseek v4 flash` |
| `novelist-writer` | 한국어 창작 산문 | Dense 또는 dense-like 창작 | `gemma 4 31b` |
| `novelist-editor` | 미시 편집과 문체 제어 | Dense 또는 dense-like 편집 | `gemma 4 31b` |
| `novelist-otaku` | 캐논 및 가지 진행의 exhaustive 검증 | 최강 추론 / 긴 문맥 | `deepseek v4 flash` |
| `novelist-loremaster` | 캐논 검색과 취합 | 추론 / 검색 / 긴 문맥 | `deepseek v4 flash` |
| `novelist-researcher` | 사실 조사와 출처 종합 | 추론 / 조사 가능 | `deepseek v4 flash` |
| `novelist-publisher` | EPUB 원본 및 패키징 | 추론 / 구조화 출력 / 코드 | `deepseek v4 flash` |
